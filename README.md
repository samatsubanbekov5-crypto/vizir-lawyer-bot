# 🏛️ Визирь — Бот для юристов

Telegram-бот для управления заявками от клиентов и координации работы юристов в сервисе "Визирь".

## 🎯 Возможности

### Для юристов:
- ✅ Регистрация с указанием специализации и контактов
- 🔔 Уведомления о новых заявках от клиентов
- 📋 Просмотр своих заявок (новые, в работе, выполненные)
- ✋ "Взять заявку" — закрепить заявку за собой
- ✅ Отметить заявку как выполненную
- ⭐ Получение оценок от клиентов
- 📊 Личная статистика (количество заявок, средняя оценка, время выполнения)

### Для администратора:
- 🏛️ **Админ-панель (мини-CRM)** — команда `/admin` (только для ID 7728619214)
- 📋 Просмотр всех заявок с фильтрами (новые, в работе, выполненные)
- 🏆 Рейтинг юристов (по количеству заявок, оценкам, времени выполнения)
- 📊 Аналитика:
  - Статистика за день/неделю/месяц
  - Распределение заявок по типам
  - Самый активный юрист
  - Средняя оценка системы
- 👨‍⚖️ Управление юристами:
  - Просмотр профилей
  - Отправка сообщений (похвала, замечания)
  - Блокировка/разблокировка
- ⭐ Просмотр оценок клиентов
- 💾 **Резервное копирование:**
  - Создание бэкапа → отправка файла в Telegram
  - Восстановление из JSON-файла

### Хранение данных:
- 📁 **SQLite** — основная БД (быстро, надёжно)
- 💾 **JSON-бэкапы** — автоматическое резервное копирование
- 🔄 Восстановление из бэкапа одной кнопкой

## 🚀 Установка и запуск

### Локально

1. **Клонируем репозиторий:**
```bash
git clone https://github.com/ваш-аккаунт/vizir-lawyer-bot.git
cd vizir-lawyer-bot
```

2. **Создаём виртуальное окружение:**
```bash
python3 -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
```

3. **Устанавливаем зависимости:**
```bash
pip install -r requirements.txt
```

4. **Создаём `.env` файл:**
```bash
cp .env.example .env
# Редактируем .env и добавляем токены
```

5. **Запускаем приложение:**
```bash
python app.py
```

Приложение будет доступно на `http://localhost:8000`

### На Render.com

1. **Создаём новый сервис на Render.com:**
   - Тип: Web Service
   - Repository: Ваш GitHub репозиторий
   - Environment: Docker

2. **Добавляем переменные окружения:**
   - `LAWYER_BOT_TOKEN` — токен бота юристов
   - `LAWYER_WEBHOOK_URL` — URL вашего сервиса (будет выдан Render)
   - `ADMIN_ID` — 7728619214
   - `CLIENT_BOT_TOKEN` — токен основного бота
   - `CLIENT_BOT_WEBHOOK_URL` — URL основного бота

3. **Добавляем диск для данных:**
   - Путь: `/data`
   - Размер: 1 GB

4. **Деплоим:**
   - Render автоматически создаст Docker образ и запустит приложение

## 📡 API Эндпоинты

### Webhook
```
POST /webhook
```
Получает обновления от Telegram

### Здоровье приложения
```
GET /api/health
```
Проверка статуса

### Резервное копирование
```
GET /api/backup
```
Скачать JSON-бэкап

```
POST /api/backup/create
```
Создать новый бэкап

```
POST /api/backup/restore
```
Восстановить из бэкапа

### Связь с ботом клиентов
```
POST /api/notify_new_request
```
Получить уведомление о новой заявке и отправить юристам

```
POST /api/request_completed
```
Отметить заявку как выполненную

```
POST /api/request_rated
```
Сохранить оценку от клиента

## 🔗 Связь с основным ботом

Бот юристов получает уведомления о новых заявках от основного бота через API:

```bash
curl -X POST https://vizir-lawyer-bot.onrender.com/api/notify_new_request \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": 123456789,
    "client_name": "Иван Иванов",
    "client_username": "ivan_ivanov",
    "description": "Нужна консультация по трудовому праву",
    "type": "консультация"
  }'
```

Когда юрист завершает заявку, основной бот получает уведомление:

```bash
curl -X POST https://vizir-bot.onrender.com/api/request_completed \
  -H "Content-Type: application/json" \
  -d '{
    "request_id": "VZ-1",
    "lawyer_id": 987654321
  }'
```

## 📊 Структура БД

### Таблица `requests`
- `id` — уникальный ID заявки (VZ-1, VZ-2, ...)
- `client_id` — ID клиента в Telegram
- `client_name` — ФИО клиента
- `client_username` — username клиента
- `description` — описание проблемы
- `type` — тип заявки (консультация, документ, жалоба, контакт)
- `status` — статус (новая, в работе, выполнена)
- `date` — дата создания
- `lawyer_id` — ID юриста, взявшего заявку
- `lawyer_name` — ФИО юриста
- `assigned_date` — дата закрепления за юристом
- `completed_date` — дата завершения
- `rating` — оценка от клиента (1-5)
- `rating_comment` — комментарий к оценке
- `rating_date` — дата оценки

### Таблица `lawyers`
- `telegram_id` — ID юриста в Telegram
- `username` — username в Telegram
- `full_name` — ФИО
- `specialization` — специализация
- `contact` — контактная информация
- `registration_date` — дата регистрации
- `requests_count` — количество взятых заявок
- `completed_count` — количество выполненных заявок
- `blocked` — заблокирован ли (0/1)
- `block_reason` — причина блокировки
- `block_date` — дата блокировки

### Таблица `ratings`
- `id` — ID оценки
- `request_id` — ID заявки
- `lawyer_id` — ID юриста
- `lawyer_name` — ФИО юриста
- `client_name` — ФИО клиента
- `rating` — оценка (1-5)
- `comment` — комментарий
- `date` — дата оценки

## 🔐 Безопасность

- Админ-панель доступна только пользователю с ID 7728619214
- Все данные хранятся локально в SQLite
- JSON-бэкапы содержат все данные и должны храниться безопасно
- Токены хранятся в переменных окружения (не в коде)
